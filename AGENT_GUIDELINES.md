# Agent Guidelines

## Operational Constraints
- **Git/Files**: No deletions or commits without explicit permission.
- **GitHub**: Do not use "co-authored by claude".
- **Banned Words**: 
    - `comprehensive` (never use in docs or commits).
    - `simplified` (never use; use `clean` or `efficient` instead).
- **Links**: No fake web links. Use real ones or omit.

## Work Standards
- **Scope**: Plans must cover the COMPLETE scope, including harsh or conflicting truths.
- **Completeness**: No "demos", "simulations", or "in a real implementation" stubs. Build the actual system.
- **Coding**: Secure by default.

## Interaction Style
- **Conciseness**: Brief, direct responses.
- **Ambition**: Extraordinary ideas.
- **Accuracy**: No omissions of critical details for the sake of positivity.
